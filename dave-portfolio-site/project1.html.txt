      <article>
        <h2>Overview</h2>
        <p>
          This project focused on migrating a fragile, manual deployment process to a
          robust CI/CD platform. The goal was to support frequent, small releases with
          confidence, while reducing downtime and deployment risk.
        </p>

        <h2>Objectives</h2>
        <ul>
          <li>Automate build, test, and deployment steps end-to-end.</li>
          <li>Support blue-green and canary deployments for safer releases.</li>
          <li>Standardize workflows for multiple applications and teams.</li>
          <li>Provide clear feedback and visibility to developers and stakeholders.</li>
        </ul>

        <h2>Solution</h2>
        <p>
          I designed a cloud-native CI/CD platform leveraging containerization and
          orchestration. Each code change triggered a pipeline that built Docker images,
          ran unit and integration tests, scanned for vulnerabilities, and deployed to
          Kubernetes using Helm charts.
        </p>
        <p>
          Promotion between environments (dev, staging, production) was controlled via
          pull requests and environment protections, ensuring traceability and auditability
          for every change.
        </p>

        <h2>Key Responsibilities</h2>
        <ul>
          <li>Architected the pipeline and environment strategy.</li>
          <li>Created reusable GitHub Actions workflows and templates.</li>
          <li>Developed Helm charts and Kubernetes manifests for services.</li>
          <li>Integrated automated testing, image scanning, and policy checks.</li>
          <li>Trained teams on using and extending the new delivery platform.</li>
        </ul>

        <h2>Impact</h2>
        <ul>
          <li>Release time reduced from multi-day windows to under one hour.</li>
          <li>Rollback capability improved via versioned Helm releases.</li>
          <li>Deployment consistency increased across all environments.</li>
          <li>Developers gained immediate feedback on code quality and deployability.</li>
        </ul>
      </article>
